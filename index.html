RegEx

A Regular Expression, or RegEx, is a pattern that can be used to describe a specific subset of text. Regular expressions use various special character codes to denote sets of characters and shortcuts to generate a searchable pattern. This tutorial will specifically cover how to utilize regular expressions in JavaScript.

Greedy vs. Lazy Quantifiers

As you may have noticed in the previous example, the regular expression found matches for each quanitfier. The results, however, were different for each quanitifier. This is because quantifiers are "greedy." Greedy quantifiers try to match at a given position and go as far forward as they can before the match fails. When it fails, it backtracks one position at a time.

Consider this "greedy" regular expression against this string.

Let's see what gets highlighted in Debuggex.

In order to avoid this behavior, you can make the quantifier "lazy," or tell the expression to repeat the minimal number of times. This means they will try to match only the next position first, and after failing, will backtrack once and then trying matching again

Note: There are also "positive lookaheads" and "negative lookaheads" that can be considered, but are outside of the scope of this tutorial. https://www.regular-expressions.info/lookaround.html

Groups

When trying to replace data using RegEx, it is possible to store the matched data in order to use it either later in the expression, or later in a string replace method, by using capturing groups.

This is my favorite part of RegEx!

Of course, Debuggex gives you information about these groups.

If you use the exec function, you can view all of the matching groups where 0 is the entire string, and 1-N are the matching groups.

The matches can be referred to as $1, $2, etc. where the number corresponds to the order in which the group matches were found. The entire match can be referenced with $.

Alternatively, you can also create "non-capturing" groups.

The Debuggex diagram illustrates the difference between these really well.
